customfield_38703 NR BB cell set TDD eCPRI (O-FH) ABIP 12cells / ABIQ 6 cells
customfield_38704 Functional Description: This feature introduces baseband cell sets for NR TDD eCPRI with open FH Interface for ABIP and ABIQ. ABIP and ABIQ capacity units can support two basic baseband cell sets per plug in card. *Following cell set is supported on ABIP with total 12 fronthaul eCPRI ports flexibly shared by both basic BB cell sets:* +Basic cell set type CB011098p-N+ * NR TDD eCPRI * Up to 6 cells of max 100 MHz cBW with 4Tx/4Rx or 2Tx/2Rx supporting 4DL-2UL or 2DL-2UL MIMO layers * No beamforming * eCPRI link (NetConf/Yang) 10GE or 25GE * Nokia recommended Open Fronthaul profile based on ORAN version 12 according CB010792 +Basic cell set type CB011098p-D+ * NR TDD eCPRI * Up to 6 cells of max 100 MHz cBW in beamforming mode supporting 4DL-2UL or 2DL-2UL MIMIO layers or non beamforming with 4Tx/4Rx or 2Tx/2Rx. * eCPRI link (NetConf/Yang) 10GE or 25GE * DoCoMo specific Open FH profile based on ORAN version 7 according CB010527 *Following cell sets are supported on ABIQ with total 9 fronthaul eCPRI ports flexibly shared by both basic BB cell sets:* +Basic cell set type CB011098q+ * NR TDD eCPRI * Up to 3 cells of max 100 MH cBW in 4Tx/4Rx or 2Tx/2Rx * No beamforming * eCPRI link (NetConf/Yang) 10GE or 25GE * Nokia recommended Open Fronthaul profile based on ORAN version 12 according CB010792 Common characteristics: * Cells with different bandwidths and or number of TX/RX streams can be mixed freely in the same BB cell set. * Mixing eCPRI RUs for O-FH (NetConf m-plane) and Nokia eCPRI RUs (with SOAP m-plane) in same BB cell set is not supported. * Two basic BB cell sets can be allocated to one ABIP or ABIQ. The combination of different basic BB cell set types in this feature is limited to those with Nokia O-FH profile and otherwise follows the rules defined in CB009181, CB009182. * BB pooling must be activated for this feature (see CB010891) The existing configuration blocks defined for Nokia eCPRI TDD RUs 4T4R can be re-used with these BB cell sets for open FH interface with Nokia profile but without Super Cell support only. Internal Information: This feature shall re-use the already existing implementation of NR TDD cell sets (valid for 4T4R RRHs) for eCPRI interface and provide exactly the same cell capacity as implemented under following features: * ABIP NR TDD up to 12 cells with max 100 MHz: CB009142 * ABIQ NR TDD up to 6 cells with max 100 MHz: CB010553 In case the cell count in above listed features will need to be reduced, the cell numbers of this feature will have to be updated accordingly as well. *Carrier BW options and O-RU support:* * This feature shall support all cBW options currently implemented for NR TDD eCPRI RUs and ported to Open FH. ( see porting feature CB009420 for ABIO which itself is ported to ABIP by CB010358) * However, as we currently don't have O-RUs supporting NR carriers BW other than 100Mhz, all cBW options beside 100 MHz need to be verified with simulator. Currently planned O-RUs for Nokia O-FH profile are: ** [CB010918:OSQA RRH 4T4R n78 160W|https://focalpoint-prod.int.net.nokia.com/fp/workspace/1314/element/39009] ** [CB012673:OSQB RRH 4T4R n78 160W|https://focalpoint-prod.int.net.nokia.com/fp/workspace/1314/element/50383] *Configuration blocks for DoCoMo specific profile* are not defined by Nokia. See also CB012774 *Inheritance of existing eCPRI configurations for Open FH under this feature:* This feature covers also the inheritance of following very basic TDD eCPRI configuration blocks for usage with O-RUs (NetConf/YANG m-plane). These configuration blocks support a 3-sector configuration with one O-RU per sector in 4Tx/4Rx or 2Tx/2Rx mode: * For ABIP ** 24R1_N_CB009487_1 for single carrier per O-RU (two config blocks per basic cell set possible) ** 24R1_N_CB009487_2 for dual carrier per O-RU (one config block per basic cell set possible) * For ABIQ ** 24R1_N_CB010498_1 - for single carrier per O-RU (one config block per basic cell set possible) ** 24R1_N_CB010498_2 - for dual carrier per O-RU (both basic cell sets of ABIQ needed) *Mixed BB cell sets on same Card:* * *Interdependency between CB011098 and CB010449:* Current assumption is that CB0010449 is implemented with lower prio in same release than CB011098. Consequently the support of mixing one basic BB cell set of 11098 (TDD eCPRI O-FH) with one basic BB cell set of 10449 (FDD NR or FDD LTE eCPRI O-FH) has to be verified and released under feature CB010449 which is implemented with lower prio. * *Interdependency between CB011098and CB011158:* Current assumption is that CB011098 will have higher prio an be implemented first. Then CB011158 will add the option to the mix two BB cell sets with different m-plane (SOAP and NetConf) in Fronthaul and thus allow to serve Nokia CPRI or eCPRI RUs and 3rd party O-FH RUs from same BB capacity card. *Common implementation for different O-FH profiles:* The architecture and implementation of these BB cell sets need to be done in a way that adaptation to different O-FH profiles will be easily possible. The scope of this feature includes implementation for Nokia recommended profile and DoCoMo customer specific profile on ABIP and on Nokia recommended profile only on ABIQ. Of course, the implementation for both BB modules shall be common, the DoCoMo profile will just not be released for ABIQ but not effort must be spent to block it. FS1TA lead: Valentin Surdu
